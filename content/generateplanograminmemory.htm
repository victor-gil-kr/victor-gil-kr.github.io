GeneratePlanogramInMemory

This command enables you to generate the specified target planograms in Space Planning based on the selected template and auxiliary planograms. The command assumes that you have already configured the template, revision template, and auxiliary planograms in the Planogram Generator Designer within Space Planning, and that you have saved the planograms to flat files.

You must open the projects that contain the specified planograms before you can use this command.

Note: If you are using this command to generate planograms but Space Planning is invisible (due to the usage of the HideApplication command), include script lines that are similar to the following to ensure that the appropriate project and planogram are selected:

Copy 
int n = 0;
foreach (JDA.Intactix.Automation.Space.Project myObject in SpacePlanning.ForAllOpenProjects())
{
if (n == 0)
{
projTemplate = myObject;
}
else
{
projTarget = myObject;
break;
}
n++;
}

If the PGType field for the template planogram indicates that it is a revision template, Planogram Generator proceeds with a revision instead of a full generation.

This command returns PGMessages, which contains the PGErrorsCount, PGSystemErrors, PGWarningsCount, PGScorePercent, and PGScoreNotes fields that are updated by Planogram Generator during the generation process.

Note: The planogram generation process runs in-memory, so system performance depends on the capabilities of your computer. If you attempt to open a large number of target planograms for generation at the same time, you may run out of available memory, experience slow planogram generation, or both. This risk is significantly higher if the planograms contain large product assortments. Therefore, it is best to open, generate, and then close each target planogram file separately.

Depending on the size of your planogram files, you can improve the performance of in-memory generation by using smaller projects instead of storing a large number of target planograms in a single project. For best performance, large-scale planogram generation should be managed on one or more batch processing servers using Planogram Generator Enterprise Edition and Category Knowledge Base.

C#

Copy 
PGMessages  GeneratePlanogramInMemory(Planogram TargetPlanogram, Planogram TemplatePlanogram, [Planogram AuxiliaryPlanogram = null], [PGAnimationLevel AnimationLevel = 0])

Parameters

Parameter

Description
TargetPlanogram

Space.Planogram. The shell of a planogram to be produced by Planogram Generator, including the fixtures to be merchandised. The target planogram becomes the final planogram produced by Planogram Generator. After the planogram has been generated, it can be copied and used as a revision template to revise the copied planogram.
TemplatePlanogram

Space.Planogram. The fully-merchandised planogram that is used as an input to the Planogram Generation process. This template provides product sequencing, orientation, and other merchandising-related information.

AuxiliaryPlanogram (optional)

Space.Planogram. A fully-merchandised planogram that is used by Planogram Generator. When generating an initial planogram, this planogram provides details on additional products that were not included in the main template, along with their sequencing, orientation, and other merchandising details. This planogram also is used to identify the anchor positions in the main template. When revising a generated planogram, the auxiliary planogram stores the actions used to perform the revision.

AnimationLevel (optional)

PGAnimationLevel. Specifies the level of detail provided during the generation process:

  • 0 (None): Does not preview the generation process. This is the default value.
  • 1 (Final): Shows only the steps involved in calculating and merchandising the final planogram.
  • 2 (Actions): Shows the result as of the last step of each action during the planogram generation process. You can select any step in the action list to see the changes made to the planogram during that step.
  • 3 (Diagnosis): Shows more detail from the planogram generation process, but excludes steps that were later rejected.
  • 4 (Rejects): Shows all details, including actions that were rejected during the process. Rejected actions include those that caused an overflow or were found not to be required to prevent an overflow.

Note: For best results, this value should be set to 0 (None) or 1 (Final), especially if the HideApplication command is used to hide Space Planning while the script is run. The remaining options consume additional memory and should be used only if you intend to view the changes applied during the planogram generation process.

Example: Generate a target planogram

Classic style scripting

Visual Basic

Copy 
Dim proj1 As Space.Project
Dim proj2 As Space.Project
Dim proj3 As Space.Project
proj1 = SpacePlanning.OpenProjectFile("C:\Template_Project.psa")
proj2 = SpacePlanning.OpenProjectFile("C:\Target_Project.psa")
proj3 = SpacePlanning.OpenProjectFile("C:\Auxiliary_Project.psa")
Dim planTemp As Space.Planogram = proj1.Planograms(0)
Dim planTarget As Space.Planogram = proj2.Planograms(0)
Dim planAux As Space.Planogram = proj3.Planograms(0)
Dim PGResults As PGMessages = SpacePlanning.GeneratePlanogramInMemory(planTarget, planTemp, planAux, PGAnimationLevel.Actions)
Dim errCount As Integer = PGResults.PGErrorsCount
Dim sysErrCount As Integer = PGResults.PGSystemErrors
Dim warCount As Integer = PGResults.PGWarningsCount
Dim scorePer As Integer = PGResults.PGScorePercent
Dim scoreNote As String = PGResults.PGScoreNotes

C#

Copy 
Space.Project proj1;
Space.Project proj2;
Space.Project proj3;
Space.Project proj2 = new Space.Project();
Space.Project proj3 = new Space.Project();
proj1 = SpacePlanning.OpenProjectFile(@"C:\Template_Project.psa");
proj2 = SpacePlanning.OpenProjectFile(@"C:\Target_Project.psa");
proj3 = SpacePlanning.OpenProjectFile(@"C:\Auxiliary_Project.psa");
Space.Planogram planTemp = proj1.Planograms[0];
Space.Planogram planTarget = proj2.Planograms[0];
Space.Planogram planAux = proj3.Planograms[0];
PGMessages PGResults = SpacePlanning.GeneratePlanogramInMemory(planTarget, planTemp, planAux, PGAnimationLevel.Actions);
int errCount = PGResults.PGErrorsCount;
int sysErrCount = PGResults.PGSystemErrors;
int warCount = PGResults.PGWarningsCount;
int scorePer = PGResults.PGScorePercent;
String scoreNote = PGResults.PGScoreNotes;
