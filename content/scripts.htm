Scripts

Create a script

  1. Open Microsoft Visual Studio.
  2. In the Start Page tab, click New Project. The New Project dialog box is displayed.
  3. Under Installed, expand one of the following folders:
    • Templates > Visual Basic > Space Automation
    • Templates > Visual C# > Space Automation

    Note: In the Visual Studio 2019 new project template filter, Space Automation is available as a platform. Select Space Automation from the Platform drop-down list and then select C# or Visual Basic for the required language template.

  4. Select an option to specify the type of script to create:

    Note: Each assembly is executed in process within Space Planning or Floor Planning. It is loaded into its own AppDomain so that it can be unloaded when the execution completes.

    • Floor BRV Script Assembly: Create a script assembly to validate the floorplans that are loaded from or saved to a Category Knowledge Base database. See Create a validation script.
    • Floor BRV Script Executable: Build a test application with a Floor Planning validation script assembly.
    • Floor Script Assembly: Create a script assembly that can be attached to the Floor Planning Tools menu.
    • Floor Script Executable: Build a test application with a Floor Planning script assembly.
    • General Script Assembly: Create a generic script assembly project in the existing solution. You must establish a reference to the new project in the existing console application for scripting. See the Visual Studio documentation for information.
    • General Script Executable: Build the required references for an automation script, including the script assembly and console application.
    • CKS Script Assembly: Create a Category Knowledge Server script assembly that can be executed with Category Batch Processing.
    • CKS Script Executable: Build a test application with a CKS script assembly.
    • Space BRV Script Assembly: Create a script assembly to validate the planograms that are loaded from or saved to a Category Knowledge Base database. See Create a validation script.
    • Space BRV Script Executable: Build a test application with a Space Planning validation script assembly.
    • Space Script Assembly: Create a script assembly that can be attached to the Space Planning Tools menu.
    • Space Script Executable: Build a test application with a Space Planning script assembly.
  5. Enter the Name for the assembly or executable. The name is also applied to the Solution name.
  6. Select the Location of the assembly or executable or click Browse to select the location.
  7. Click OK. The Space Automation New Assembly dialog box is displayed.
  8. Enter the name of the assembly in the Script Assembly field.
  9. Enter the name of the script in the Script File field.

    10. Notes:

    • The Script Assembly and Script File names should be different.
    • The console project should be set as "startup Project" before you run the script.
  10. Click OK. If you selected to create an executable, a console application is created and the script is added to that console. If you selected to create an assembly, the script is created, but you have to add it to a console application manually.
  11. If the Space Automation Commands window is not displayed, display the Command Explorer.
  12. Add commands from the Command Explorer to the appropriate area of the script file.

    13. Note: The Command Explorer window displays all commands that can be used with Space Automation Professional. You can double-click commands in the Command Explorer window to add them to your script. If the command requires additional parameters, a dialog box may be displayed to enable you to specify those parameters. When a dialog box is displayed, an asterisk identifies required fields.

Add a new script file to a script assembly

The same console application can be used to run different script assemblies. You can also create multiple script files within an existing script assembly to perform various tasks.

  1. Create a script.
  2. Select the script assembly to which you want to add another script file.
  3. Right-click the script assembly, then select Add > New Item. The Add New Item dialog box is displayed.
  4. Under Installed > Templates > Other Languages > Visual Basic, select Space Automation.
  5. Select the General Script File template.
  6. Enter the Name of the script file, then click Add.
  7. If the Space Automation Commands window is not displayed, display the Command Explorer.
  8. Add commands from the Command Explorer to the appropriate area of the script file.

Note: The Command Explorer window displays all commands that can be used with Space Automation Professional. You can double-click commands in the Command Explorer window to add them to your script. If the command requires additional parameters, a dialog box may be displayed to enable you to specify those parameters. When a dialog box is displayed, an asterisk identifies required fields.

View script errors

If errors occur when you run a script, Space Automation Professional generates an error log to store the error information. The error log is named SpaceAutomation_MMDDYYYYHHMMSS.log (where MMDDYYYYHHMMSS refers to the date and time the file was created) and is located in the folder in which Space Automation Professional is installed.

Error handling

Space Automation Professional enables you to add your own structured error handling code by using Try-Catch exception handling. For example:

Copy 
Module Module1
    Sub Main()
        Dim MyScript As New Sample()
        Try
            MyScript.Run()
        Catch ex As Exception
            Dim errors = MyScript.GetErrors()
            Debug.Print("List of Errors : ")
            For Each keyValuePair As KeyValuePair(Of String, List(Of String)) In errors
                For Each errString As String In keyValuePair.Value
                    Debug.Print(errString)
                Next
            Next
        End Try
    End Sub
End Module

See the appropriate language guide for more information.

You can also interrogate the script method GetErrors() to obtain detailed information on the errors that are reported. GetErrors() is a script level method that returns a .Net Dictionary collection object that contains any errors generated during script execution.

These errors also may be reported in the error log. See View script errors.

Deploy scripts to a runtime environment

If you create a general project that creates a console.exe file and Space Automation Professional assembly, you can deploy the DLL or executable to an environment where Space Automation Profession is not installed.

Note: If your script uses database commands to work with data in a Category Knowledge Base database, you should configure database settings in SpaceAutomationpro.config or <Assembly Name>.dll.config and then deploy the file to each runtime environment.

In the development environment, up to 10 different database connections can be configured and the developer can set any of those connections as the active (default) connection. In a runtime environment, when using SpaceAutomationPro.config, only the default database connection can be used, regardless of the number of connections that are specified.

You can use SAProConfigEditor.exe to update SpaceAutomationpro.config or <Assembly Name>.dll.config with the database settings to be used for runtime environments.

To deploy the script executable (console.exe), you must copy all dependent files, including assembly.dll, to the runtime environment. Also, you must copy the JDA.Intactix.Automation.CommandPane.dll file from the Space Automation Professional installation folder in the development environment to the general executable (console.exe) folder in the runtime environment.

The following processes apply based on the type of application that has been created:

  • If the application has been created as a general script executable, the user can run it by performing one of the following tasks:
    • Double-click Console.exe.
    • Attach Console.exe to the Space Planning or Floor Planning Advanced ribbon.
    • Add Console.exe to Prospace.ini or Profloor.ini as MenuX.

    • Note: To update the database settings, the user must update the database details in the Console.exe.config file that exists in the folder that contains Console.exe.

  • If the application has been created as a Space Script Executable or Floor Script Executable, the user can run it by performing one of the following tasks:
    • Double-click Console.exe.
    • Attach Console.exe to the Space Planning or Floor Planning Advanced ribbon.
    • Attach the script assembly executable file to the Space Planning or Floor Planning Advanced ribbon.
    • Add Console.exe to Prospace.ini or Profloor.ini as MenuX.
    • Add the script assembly executable file to Prospace.ini or Profloor.ini as MenuX.

    • Note: To update the database settings, the user must update the database details in the <Assembly_Name>.dll.config file that exists in the folder that contains Console.exe. You also must use the SelectDatabase command for database-related transactions; otherwise, the script executable or assembly relies on the client application's active database connection.

  • If the application has been created as a Space Script Assembly or Floor Script Assembly, the user can run it by performing one of the following tasks:
    • Attach the script assembly DLL to the Space Planning or Floor Planning Advanced ribbon.
    • Add the script assembly DLL to Prospace.ini or Profloor.ini as MenuX.

    • Note: To update the database settings, the user must update the database details in the <Assembly_Name>.dll.config file that exists in the folder that contains <Assembly_Name>.dll. You also must use the SelectDatabase command for database-related transactions; otherwise, the script executable or assembly relies on the client application's active database connection.

Notes:

  • When the script is compiled, the log file path in Console.exe.config is set to the same log file path as in the SpaceAutomationPro.config file in the development environment. The log file path can be specified manually in Console.exe.config for the runtime environment, and also can be accessed through a script assembly. Separate logging logic should be implemented for the script assembly if the log file path is not configured in assembly.dll.config.

  • In the development environment, multiple database connections can be configured. A script can use any one of the configured connections by specifying the connection in the SelectDatabase command. When config files are created for the runtime deployment as part of the script compilation, only the default database connection is included. Any attempts to use another database connection when the script is executed in the runtime environment will fail. To use the active connection specified in the config file, use the SelectDatabase command with the Database parameter blank.

Use the database configuration editor

When you deploy a script assembly to a runtime environment, the SpaceAutomationPro.config, <Assembly Name>.dll.config, or console.exe.config configuration file contains the connection details that are required to run a script that interacts with a Category Knowledge Base database.

You can use the SAProConfigEditor.exe utility, located in the \Space Automation Pro\Utilities\SAProConfigEditor folder on the installation media, to change the database credentials on a client machine without having Space Automation Professional installed.

  1. Copy SAProConfigEditor.exe from the SAProConfigEditor folder on the installation media to the client folder where the script assembly and SpaceAutomationPro.config files are deployed.
  2. Double-click SAProConfigEditor.exe. The Space Automation Professional Database Configuration utility is displayed.
  3. Click the ellipsis button next to the Configuration file to update field.
  4. Select the SpaceAutomationPro.config or <AssemblyName>.dll.config file to update, and then click Open.
  5. Under Connection type, select Direct or Web.
  6. If you selected Direct, under Database type, select SQL Server or Oracle and then specify the database login credentials.
  7. If you selected Web, specify the URL, User name, and Password to connect to the database.
  8. To verify the specified database credentials, click Test Connection. A message indicates if the connection was successful. Click OK.
  9. Click Save to apply the credentials to the .config file.
  10. Click Close to exit the utility.
