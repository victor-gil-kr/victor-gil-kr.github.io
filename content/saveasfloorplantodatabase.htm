SaveAsFloorplanToDatabase

This command enables you to save the active floorplan into a Category Knowledge Base database. You can use this command to:

  • Save floorplan changes over an existing database floorplan
  • Save floorplan changes as a copy to a different location in the floorplan hierarchy (the original floorplan remains in its original location)
  • Save a new floorplan to the database from a Floor Planning flat file
  • Save a new version of the floorplan to a specific location

To save floorplans to their original location in the database, you should use the SaveFloorplanToDatabase command. Otherwise, you will create new copies of the saved floorplans.

To save the active floorplan as a version of another floorplan in the database, you should use the SaveAsFloorplanVersionToDatabase command.

Notes: 

  • You must include the SelectDatabase command before this command to select the database to be accessed by your script.
  • This command is supported only when Space Automation Professional is configured to use a direct database connection.
  • This command is not needed in a Business Rule Validation script that is used to validate floorplans saved from Floor Planning.

SaveAsFloorplanToDatabase(string NodeKey, int Status, bool AddNewPlanograms, bool UpdatePlanograms, bool SaveAsVersion, [int? DBKey = null], [string EffectiveFromDate = null], [string EffectiveToDate = null])

Parameters

Parameter

Description
NodeKey

String. The node in the floorplan or planogram hierarchy into which to save the floorplan or planogram. When you select a node in the hierarchy, the field is filled in automatically. The field value is formatted based on hierarchy information in the Category Knowledge Base database, so it should not be modified.

Status

Integer. The status for the planograms or floorplans being saved back into the database. If specifying the Status parameter value, this value must be presented as its numeric equivalent. For example, type "1" to save the floorplan or planogram with a Live status. If setting a field value for an object's Status field, enter the text string ("Live"). See "Status" in the Category Knowledge Base OnLine Expert.

If you have enabled Object Versioning restrictions in your Category Knowledge Base database, you cannot save floorplans or planograms to the database with a status of Live. See "Object versioning overview" in the Category Knowledge Base OnLine Expert.

If you have enabled Lifecycle Management for planograms or floorplans in Category Knowledge Base, this parameter is ignored.

New planograms and floorplans are saved with the Work in Progress status automatically.

AddNewPlanograms

Boolean. Specifies whether to add new planograms that exist in the floorplan or project when saving to a Category Knowledge Base database. If False is selected, sections of the new planograms are also removed from the floorplan or project saved to the database.

UpdatePlanograms

Boolean. Specifies whether to update existing planograms in the floorplan or project being saved. If False is selected, planogram data changes in the floorplan or project being saved are lost.

SaveAsVersion

Boolean. Specifies whether to save the planogram or floorplan as a version of the original planogram or floorplan to which changes were made.

  • If Yes is selected, the planogram or floorplan is saved as a version of the original planogram or floorplan, and is linked to that object.
  • If No is selected, the floorplan or planogram is saved as a "master" object, with no association to the original floorplan or planogram.
  • If Yes is selected and the original floorplan or planogram has a status of Live, you must save the new version with a status other than Live to correspond to the requirements of Object Versioning. See "Object versioning overview" in the Category Knowledge Base OnLine Expert.

DBKey (optional)

Integer. The optional database key for the floorplan or planogram that is overwritten with changes in the active floorplan or planogram. If you do not want to replace an existing planogram, you should not specify a DBKey. You should also not assign a DBKey if saving planograms to the database within a ForDatabaseFloorplans or ForDatabasePlanograms loop, because doing so causes all files to be saved over each other to the assigned DBKey.

EffectiveFromDate

String. The date the floorplan or planogram is to become Live. If a versioning schedule is set up in Console (or in an external scheduler for Oracle databases), this date is used to determine when to cycle the floorplan or planogram to the Live status. If you have enabled Lifecycle Management for planograms or floorplans in Category Knowledge Base, this parameter is ignored. You must use the Planogram Lifecycle page in Data Manager to specify the appropriate Date to Pending and Date to Live dates for each planogram or floorplan.

EffectiveToDate

String. The date the floorplan or planogram is to become Historic. If a versioning schedule is set up in Console (or in an external scheduler for Oracle databases), this date is used to determine when to cycle the floorplan or planogram to the Historic status. If you have enabled Lifecycle Management for planograms or floorplans in Category Knowledge Base, this parameter is ignored. You must use the Planogram Lifecycle page in Data Manager to specify the appropriate Date to Pending and Date to Live dates for each planogram or floorplan.

Example: Save a new database floorplan

Classic style scripting

Visual Basic

Copy 
FloorPlanning.OpenProjectFile("C:\Program Files (x86)\JDA\Intactix\Samples\Floor Planning Floorplans\Sample - Store 001.PFA")
FloorPlanning.SelectDatabase("SampleDatabase|(local)|user1")
FloorPlanning.SaveAsFloorplanToDatabase("1", 1, True, True, False, 1, "05/21/2013", "06/21/2013")

Copy 
FloorPlanning.OpenProjectFile(@"C:\Program Files (x86)\JDA\Intactix\Samples\Floor Planning Floorplans\Sample - Store 001.PFA");

FloorPlanning.SaveAsFloorplanToDatabase("1", 1, true, true, false, 1, "05/21/2013", "06/21/2013");
