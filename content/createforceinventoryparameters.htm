CreateForceInventoryParameters

This command enables you to create a list of parameter settings for use when performing a Force Inventory on planograms in the specified projects. The parameters for this command correspond to the settings available on the Space Planning Force Inventory dialog box. After the Force Inventory Parameter list has been created, it can be used by inserting the Force Inventory command and selecting the appropriate list. See "Force Inventory overview" in the Space Planning OnLine Expert for information on the Force Inventory settings.

Syntax

Visual Basic

Copy 
CreateForceInventoryParameters(_
    ListName As String,_
    PositionFrom As SpaceFIPositionFrom,_
    Goal As SpaceFIGoal ,_
    Space As SpaceFISpace,_
    Measure As SpaceFIMeasure,_
    Formula As String,_
    FIScale As Integer,_
    ManualFacings As SpaceFIManualFacings,_
    StayWithinMinMax As Boolean,_
    MinimumLimit As SpaceFIMinMaxLimit,_
    MaximumLimit As SpaceFIMinMaxLimit,_
    XAdjustable As Boolean,_
    YAdjustable As Boolean,_
    ZAdjustable As Boolean_
)

C#

Copy 
CreateForceInventoryParameters(string ListName, [SpaceFIPositionFrom PositionFrom = null], [SpaceFIGoal Goal = null], [SpaceFISpace Space = null], [SpaceFIMeasure Measure = null], [string Formula = null], [int FIScale = 2], [SpaceFIManualFacings ManualFacings = null], [bool StayWithinMinMax = false], [SpaceFIMinMaxLimit MinimumLimit = null], [SpaceFIMinMaxLimit MaximumLimit = null], [bool XAdjustable = true], [bool YAdjustable = false], [bool ZAdjustable = false])

Parameters

Parameter

Description
ListName

String. The unique name of the list that contains the Force Inventory parameters. This name is selected in the Force Inventory command.
PositionFrom (optional)

SpaceFIPositionFrom. Specifies the object on which to adjust positions:

  • 0: Adjust positions on the selected objects
  • 1: Adjust all positions on the planogram
  • 2: Adjust all positions in the entire project.

In the command's dialog box, you can select the appropriate option from the Adjust positions from drop-down list.
Goal (optional)

SpaceFIGoal. The goal to be used to adjust facings on the planogram:

  • 0: Resets selected positions to one facing each.
  • 1: Adds or removes facings to match the Target Inventory values. If it is not possible to match the Target Inventory, the value is exceeded. Fixtures may be overfilled and the total capacity of the planogram may be exceeded. This option best represents the amount of space needed to carry all products at their Target Inventory values.
  • 2: Increases or decreases the number of facings for positions by a specified factor. For example, a factor of 2.0 doubles the number of facings. Fixtures may be overfilled and the total capacity of the planogram may be exceeded.
  • 3: Adds facings to fill the spaces at the ends of fixtures, without overfilling the fixtures. Facings are adjusted in proportion to the number of existing facings, so a position with a high number of facings is expanded more than a position with fewer facings.
  • 4: Adds facings to fill the spaces at the ends of fixtures, without overfilling the fixtures. The most under-stocked products (those with the lowest ratio of Capacity to Target Inventory) are adjusted first.
  • 5: Adds facings to fill the cubic space for each fixture. Facings are added to balance some measure per space ratio, such as Cubic Space to Sales. In this example, positions on each fixture are reviewed to determine the ratio of Cubic Space to Sales. The position with the lowest ratio is expanded first. After each adjustment is made, the positions are reviewed to determine the next position to be adjusted.
  • 6: Adds facings to fill out the total cubic space available in the planogram. The total cubic space is the total cubic space of the fixtures on the planogram, ignoring positions on the fixtures. This option may overfill some fixtures while others remain half full. Therefore, some positions may need to be rearranged manually. Facings are adjusted in proportion to the number of existing facings, so a position with a high number of facings is expanded more than a position with fewer facings.
  • 7: Adds facings to fill out the total cubic space available in the planogram. This option may overfill some fixtures while others remain half full. Therefore, some positions may need to be rearranged manually. The most under-stocked products are adjusted first.
  • 8: Adds facings to fill out the total cubic space available in the planogram. This option may overfill some fixtures while others remain half full. Therefore, some positions may need to be rearranged manually. Facings are added to balance some measure per space ratio (such as Cubic Space to Sales). In this example, positions on the planogram are reviewed to determine the ratio of Cubic Space to Sales. The position with the lowest ratio is expanded first. After each adjustment is made, the positions are reviewed to determine the next position to be adjusted.
  • 9: Adds or removes facings in an attempt to balance the ratio of space to measure for each product, without overfilling fixtures. For example, if the current average of Sales per Cubic is $4 per cu ft, facings are added or removed to try to match this average.
  • 10: Adds or removes facings in an attempt to balance the ratio of space to measure for each product, without exceeding the total cubic space on the planogram. For example, if the current average of Sales per Cubic is $4 per cu ft, facings are added or removed to try to match this average. This option may overfill some fixtures while others remain half full, so some positions may need to be manually rearranged afterwards.
  • 11: Adds or removes facings in an attempt to balance the ratio of space to measure for each product. For example, if the current average of Sales per Cubic is $4 per cu ft, facings are added or removed to try to match this average. Some fixtures and the entire planogram may be overfilled when using this option.
  • 12: Removes positions with zero Target Inventory.
  • 13: Removes positions with zero values for the selected measure (such as Sales).
Space (optional)

SpaceFISpace. Specifies how space is defined when you select a Force Inventory goal that refers to space:

  • 0: Units
  • 1: Facings
  • 2: Width
  • 3: Area
  • 4: Cubic Space

In the command's dialog box, you can select the appropriate option from the Where space is drop-down list.

Measure (optional)

SpaceFIMeasure. Specifies the measure that is used when the selected Force Inventory goal refers to a measure:

  • 0: Target Inventory
  • 1: Sales
  • 2: Profit
  • 3: ROII
  • 4: Movement
  • 5: Formula

In the command's dialog box, you can select the appropriate option from the ...and measure is drop-down list.

Note: The measure fields refer to Performance-level fields by default. They may refer to the Product-level fields depending on the Use Performance Price setting on the Performance tab of the Project Properties dialog box in Space Planning.

Formula (optional)

String. The formula that defines the measure to be used by the goal. This option applies when Measure (... and measure is) is set to 5 (Formula).

Scale (optional)

Integer. The scaling factor by which to adjust facings. For example, to double the number of facings, enter 2. To reduce the number of facings by half, enter 0.5.

ManualFacings (optional)

SpaceFIManualFacings. Specify the method used to adjust the number of units wide, high, or deep for positions where the number of units is manually determined:

  • 0: Increase, but do not decrease, the number of units.
  • 1: Decrease the number of units for manually determined positions if the fixture on which it resides is overfilled.
  • 2: Adjust the number of units for positions.
  • 3: Set all positions where the number of units is manually determined to one, then increase to the appropriate number of units.
  • 4: Do not adjust positions if the number of units is manually determined.

In the command's dialog box, you can select the appropriate option from the Manual facings/units drop-down list.

StayWithinMinMax (optional)

Boolean. Specifies whether to stay within manual minimum and maximum limits when adjusting the number of units wide, high, or deep.

In the command's dialog box, you can select Stay within min/max limits to enable this option.

MinimumLimit (optional)

SpaceFIMinMaxLimit. Specifies how to adjust the manual minimum limits for positions set to Fill Available Space:

  • 0: Retain the minimum limits.
  • 1: Remove the minimum limits.
  • 2: Adjust the minimum limits when required.

In the command's dialog box, you can select the appropriate option from the Minimum limit drop-down list.

MaximumLimit (optional)

SpaceFIMinMaxLimit. Specifies how to adjust the manual maximum limits for positions set to Fill Available Space:

  • 0: Retain the minimum limits.
  • 1: Remove the minimum limits.
  • 2: Adjust the minimum limits when required.

In the command's dialog box, you can select the appropriate option from the Maximum limit drop-down list.

XAdjustable (optional)

Boolean. Specifies whether the X dimension can be adjusted for positions.

In the command's dialog box, you can select Adjust facings wide to enable this option.

YAdjustable (optional)

Boolean. Specifies whether the Y dimension can be adjusted for positions.

In the command's dialog box, you can select Adjust units high to enable this option.

ZAdjustable (optional)

Boolean. Specifies whether the Z dimension can be adjusted for positions.

In the command's dialog box, you can select Adjust units deep to enable this option.

Example: Force inventory on the planogram

Classic style scripting

Visual Basic

Copy 
SpacePlanning.OpenProjectFile(“C:\Program Files (x86)\JDA\Intactix\Samples\Space Planning Planograms\Sample - Coffee Makers.psa”)
SpacePlanning.CreateForceInventoryParameters("MeasureNoLimits", SpaceFIPositionFrom.Planogram, SpaceFIGoal.AverageSpaceToMeasureNoLimits, SpaceFISpace.Facings, SpaceFIMeasure.Movement)
SpacePlanning.ForceInventory("MeasureNoLimits")

C#

Copy 
SpacePlanning.OpenProjectFile(@"C:\Program Files (x86)\JDA\Intactix\Samples\Space Planning Planograms\Sample - Coffee Makers.psa");
SpacePlanning.CreateForceInventoryParameters("MeasureNoLimits", SpaceFIPositionFrom.Planogram, SpaceFIGoal.AverageSpaceToMeasureNoLimits, SpaceFISpace.Facings, SpaceFIMeasure.Movement);
SpacePlanning.ForceInventory("MeasureNoLimits");
