Event support

The following events are supported by Space Automation Professional:

public enum SpaceEvents

{

    ProjectClose = 10,

    FileOpen = 19,

    FileClose = 20,

    FileSave = 21,

    FileSaveAs = 22,

    FileOpenFromDatabase = 23,

    FileSaveToDatabase = 24,

    FilePrint = 27,

    FileBatchPrint = 28,

    FileExit = 33,

    FileSaveAsToDatabase = 35,

    FileOpenFixtureLibraryFromDatabase = 36,

    EditAddFromLibraryProduct = 37,

    EditAddFromLibraryFixture = 38,

    FileOpenDatabaseProductLibrary = 41,

    ProjectSave = 42,

    TableFileOpen = 43,

    BatchPrintFailed = 44,

    ProjectOpen = 51,

    ProjectOpenFromDatabase = 52,

}

This section describes how to support events in Space Automation Professional scripts.

  1. Start Visual Studio 2015 or later.
  2. Create a Space Planning library project of type Space Script Assembly.
  3. Select Project > Add References.

  4. Under the COM tab, add a reference to the IXInformer interop shared component available in the common folder of the installation drive. For example: 

    C:\Program Files (x86) \Common Files\JDA\Application Files\IXInformer.InterOp.dll

  5. Declare the following namespaces at the using section:

    Copy 
    using IXInformer;

  6. Add the following in the general declarations section of a class:

    Copy 
    private IXInformer.NotifyClass _notifyClass;

    Note: The Notify object from IXInformer will broadcast event notifications from the active instance of Space Planning to the calling add-on. This call enables the Space Automation Professional script assembly to determine the supported events.

    Initialize the notify class at the parameterized constructor:

    Copy 
    public SpaceMenuClass(int nSpaceOrFloor): base(nSpaceOrFloor)
    {
    _notifyClass = new NotifyClass();
    _notifyClass.BeforeEvent += new INotification_BeforeEventEventHandler(Notify_BeforeEvent);
    _notifyClass.AfterEvent += new INotification_AfterEventEventHandler(Notify_AfterEvent);
    }

    This code accesses the ixinformer notify object of Space Planning.

  7. Add the following methods to handle the Space Planning events:

    Copy 
    private void Notify_BeforeEvent(int lEventID, object pApplication, out int plAction, int lParam) {}
    private void Notify_AfterEvent(int lEventID, object pApplication, int lParam) {}

    Note: The Before event passes control to the Space Automation Professional script before executing the calling event. Similarly, the After event passes control to Space Automation Professional after the calling event has completed.

  8. Add the following to Notify_BeforeEvent:

    Copy 
    plAction = 0;
    switch ((SpaceEvents)lEventID)
    {
    case SpaceEvents.FileOpenFromDatabaseEvent:

    Notify_BeforeEvent is called by Notify after the Space Planning event identified by lEventID has occurred.

    Note: Set plAction parameter to 1 to take over a Space Planning event.

Restrictions

  • Events are only supported as a menu assembly or executable. They are not supported in Business Rule Validation, Category Knowledge Base (CKS), or non-interactive Space Planning scripts.
  • CreateObject("ProSpace.Application") will create a new instance of Space Planning and events will be connected to the new instance. Use caution while creating new instances of Space Planning.

  • It is not recommended to have COM API calls to Space Planning directly with this event through Space Automation Professional enumerations. This may lead to memory management problems that eventually might cause Space Planning to become unstable.

    If COM API calls are part of the Space Automation Professional script, ensure that you release the objects in logical order for better memory management.
