Get started

Activate Space Automation Professional

  1. Start Visual Studio.
  2. Perform one of the following tasks:

    • If you are activating for the first time, select Space Automation > Command Explorer in the menu bar.

      Note: For Visual Studio 2019, select Extensions > Space Automation -> Command Explorer.

    • To change an existing activation, click Change Activation at the bottom of the Command Explorer.

    The Space Automation Pro Activation dialog box is displayed.

  3. Enter the User Name and Activation Code provided by Customer Support.
  4. Click Activate.

Note: If you change an existing activation, you should recompile all solutions based on the new activation information.

Configure Space Automation Professional

You can modify various settings that control how Space Automation Professional scripts are run.

  1. To configure the settings in the application:

    1. Select Space Automation > Configuration or click in the Command Explorer.

      Note: For Visual Studio 2019, select Extensions > Space Automation -> Configuration.

    2. Under Launch with Space Automation Professional, select check boxes to specify whether to open Space Planning or Floor Planning automatically when you run Space Automation Professional scripts.
    3. If you work with data that is stored in a Category Knowledge Base database:
      • To log into a Category Knowledge Base database automatically when you open Space Automation Professional, under Database integration, select Log into Category Knowledge Base on application launch.

      • Select an option to specify whether to enable a direct database connection or a web-enabled connection to the Category Knowledge Base database.

        Notes: 

        • The selection made here must match the selection made in Space Planning.
        • If you change the database connection type, you must restart Visual Studio to apply the change.
        • If you select the web-enabled database connection, your scripts cannot use any of the Floor Planning database commands.
      • Click Database Configuration to configure database logins.
    4. Under Log file, in the Path field, enter the location in which you want to store the Space Automation Professional log files. If this path is not specified, the log files are stored in the folder in which Space Automation Professional is installed.
    5. Click OK.
  2. To configure the settings in the external configuration file:
    1. In Windows Explorer, navigate to the folder in which Space Automation Professional is installed. For example, the default location is C:\Program Files (x86)\JDA\Intactix\Space Automation Professional.
    2. Open SpaceAutomationPro.config in an XML or text editor.
    3. To specify the location of error log files that are generated when scripts are run, under Application Settings, locate the LogFilePath tag, and then enter the path in which log files should be stored.

      4. Notes:

      • If a LogFilePath value is not specified, the log files are stored in the folder in which Space Automation Professional is installed.
      • Log files are named SpaceAutomation_MMDDYYYYHHMMSS.log, where MMDDYYYYHHMMSS refers to the date and time the file was created.
      • If you have upgraded from a previous version of Space Automation Professional and this setting is not included in your SpaceAutomationPro.config file, add the following text to the file before you specify the value:
        Copy
        <configuration>
          <configSections>
            <sectionGroup name="ApplicationSettings">
              <section name="applicationSetting" type="JDA.Intactix.Configuration.NameValueSection, JDA.Intactix.Configuration"/>
            </sectionGroup>
            ………..
          </configSections>
          <jda.intactix.ikb>
            ………..
          </jda.intactix.ikb>
          <ApplicationSettings>
            <applicationSetting>
              <add name="LogFilePath" value=""/>
              <add name="StartSpacePlanning" value="True" />
              <add name="StartFloorPlanning" value="true" />
              <add name="LogIntoCKB" value="false" />
            </applicationSetting>
          </ApplicationSettings>
        </configuration>
    4. Save and close SpaceAutomationPro.config.

Configure database logins

You can specify login information for up to ten different Category Knowledge Base databases, and you can specify the default database that you use most often. These settings are used by the SelectDatabase command to specify the appropriate database when using scripts to process objects that are stored in the database.

If you do not configure logins for your databases or log into a Category Knowledge Base database and the SelectDatabase command includes database details, the remaining database commands in scripts cannot be run. If you use the SelectDatabase command without specifying the database parameters, the script will rely on the active database connection in Space Planning or Floor Planning.

  1. Configure Space Automation Professional to specify whether the login information refers to a direct or web-based connection to the database.

  2. Under Database integration in the Configuration dialog box, click Database Configuration. The CKB Database Configuration or Web Configuration dialog box is displayed.
  3. Perform one of the following tasks:
    • If you selected a Direct connection to the database:
      1. Click the ellipsis button to the right of the first available Database Name field. The Login dialog box is displayed.
      2. Log into a Category Knowledge Base database. The User Name and Database Name fields are filled in based on the specified settings.

    • If you selected a Web connection to the database:

      1. Enter the URL of the connection to the database in the first available URL field.
      2. Click the ellipsis button to the right of the URL field. The Login dialog box is displayed.
      3. Enter the Login Name and Password associated with the database, and then click OK.
  4. Configure other database logins. You can click Remove to the right of a database configuration to remove that database from the list. You can also click Clear All to remove all database settings.
  5. Select the Default option to the left of the database that you work with most often. This enables you to log into the selected database when Space Automation Professional is started, and prevents you from having to log into a database each time a database command is added to a script.
  6. Click OK.

Log into a Category Knowledge Base database

To create scripts to modify objects stored in a Category Knowledge Base database, you must first log into the appropriate database. This enables the controls associated with the database-related commands to display the appropriate data from the database.

Note: To specify the database type used by the Category Knowledge Base database, you must open IKBConfigEditor.exe in the Program Files (x86)\JDA\Intactix\Space Automation Professional folder. This task will synchronize Space Automation Professional with the database. By default, the file reflects the Server Type specified in the Registry under HKLM\Software\Intactix\pro/fusion\3.0\Database.

  1. If the Space Automation Commands window is not displayed, display the Command Explorer.
  2. Click . The Login dialog box is displayed.
  3. Type your user name in the Login Name field.
  4. Type your password in the Password field.

    5. Note: If the database administrator configured your user account properly, you can select the Use Windows NT Authentication check box to use your standard Windows NT login rather than using a separate Login Name and Password.

  5. If you are a SQL Server user, select the server from the Server drop-down list. If you are an Oracle user, type the necessary host string in the Host String field.
  6. Select the database from the Database drop-down list.
  7. Click OK.

Work with database queries

You can define queries to extract data from a Category Knowledge Base database. You can then execute the query to act on the extracted data.

Create a query

  1. If the Space Automation Commands window is not displayed, display the Command Explorer.

  2. Click . The Database Queries window is displayed.
  3. Perform one of the following tasks:
    • To create a new query, click Create.
    • To create a query based on the settings of an existing query, select the existing query and then click Copy.

    The Query Builder window is displayed.

  4. In the Query name field, enter a unique name for the query.

    Note: If the query will be shared with other users, the query name must be unique when compared to all other shared queries. A private query can use the same name as a shared query, but you cannot create two private queries with the same name.

  5. In the Description field, enter a description of the query.

  6. From the Application drop-down list, select the application whose data you want to query. To create a custom query for any pre-defined Category Knowledge Base database table instead of selecting the application data fields, select Custom.

    Notes:

    • You can select the Custom option only if you are using a direct database connection. If you are using a web database connection, you cannot create a custom query, but you can refer to an existing custom query created by another user.
    • When this option is selected, only the Where query field is enabled, so you can specify the SQL text.
    • For custom queries, you should supply a column alias if a function is used on a particular column.
    • To create a query for a user-defined table in the database, proper database permissions must be granted to create and run queries based on the custom table.

  7. From the Object type drop-down list, select the object whose data you want to query. A message is displayed to confirm the object change. Click Yes to accept the message.

  8. To allow the query to be used and modified by other database users, from the Query shared drop-down list, select Yes.

  9. Select the fields to be returned by the query:

    1. Select fields in the Available fields list.

    2. Click to add the fields to the Selected fields list.

  10. Under Where clause, build a Where statement to specify the objects whose data will be extracted by the query:

    1. From the Field drop-down list, select the field to review when querying the database.

    2. From the Operator drop-down list, select an option to specify how to compare the selected field.

    3. In the Value field, enter or select the field value to use to identify objects. The query will include data for objects that meet the Operator and Value criteria for the selected field.

    4. To enable users to specify field values when they use the query in a script, select the Parameter check box instead of entering a field value. This action adds "'%ParamString%'" (for text fields) or "'%ParamNumber%'" (for numeric fields) to the Where clause.

      When a user adds the Execute Query command to a script and selects this query, the script writer can specify the parameter values to associate with the query. The script writer can specify values manually or use variables and functions to set the value dynamically. Space Automation Professional will then use the specified values to identify the objects whose data will be extracted by the query.

      Note: You can add up to eight dynamic parameters to a single query. If you use dynamic parameters instead of field values in the query, you should modify the query's Description field to identify each parameter, in order, and the data that must be selected for the parameter. Modifying the description will make it easier to identify the parameter values to be specified when users add the Execute Query command to their scripts.

    5. Click Add. The rule is added to the Where query field.

  11. To add another filter to the Where clause, click And. To add an alternate filter to the Where clause, click Or. The selected keyword is added to the clause. These options enable you to build a query based on multiple conditions.

  12. If And or Or is added to the clause, repeat Step 10 to include additional statements in the clause.

    Notes:

    • If multiple And or Or statements are added to the clause, you can enclose statements in parentheses to specify the order in which each statement should be reviewed.
    • Only rules for the selected object type can be added to the Where clause. You cannot add rules for multiple object types to a single Where clause.

  13. If you selected Custom from the Application drop-down list, enter the SQL statement in the Where query field.

    Note: You are limited to entering only simple SELECT queries, such as the following:

    • select Count,DBUser from ix_spc_planogram Group by DBUser
    • select Name from ix_spc_planogram order by Name

    The system does not allow any queries that create, modify, or delete data or database objects or complex queries such as the following:

    Select Name from ix_spc_planogram where DBKey IN(select DBParentPlanogramKey from ix_spc_position)

  14. Click Check Syntax to validate the query, then correct the query, as needed.

  15. Click OK. The query is stored in the Category Knowledge Base database and can be used in Space Automation Professional scripts.

Modify a query

Note: You can only modify database queries that you have created.

  1. If the Space Automation Commands window is not displayed, display the Command Explorer.

  2. Click . The Database Queries window is displayed.

  3. Select a query in the Database Queries list.

  4. Click Edit. The Query Builder dialog box is displayed.
  5. Modify the query settings. See Create a query.

Delete a query

Note: You can only delete database queries that you have created. If the query is shared and another user attempts to run a script containing the query, the user is warned that the query no longer exists.

  1. If the Space Automation Commands window is not displayed, display the Command Explorer.

  2. Click . The SQL Builder window is displayed.

  3. Select a query in the SQL Builder list.

  4. Click Delete. A confirmation message is displayed.
  5. Click Yes.
